# Data Dictionary

Документ описывает основные таблицы, поля и связи, которые используются в проекте **Credit Portfolio & Risk Analytics**.

Модель данных отражает путь клиента в кредитном процессе:

клиент → заявка → выданный кредит → платежи

---

## 1. Таблица `clients`

Таблица содержит информацию о клиентах. Используется для клиентской сегментации и анализа риска.

| Поле | Тип данных | Описание |
|---|---|---|
| `client_id` | integer | Уникальный идентификатор клиента. Первичный ключ. |
| `birth_date` | date | Дата рождения клиента. Может использоваться для расчёта возраста. |
| `gender` | text | Пол клиента. |
| `city` | text | Город клиента. |
| `registration_date` | date | Дата регистрации клиента. |
| `income` | numeric | Доход клиента. Используется для анализа клиентского профиля и риска. |
| `employment_type` | text | Тип занятости клиента: например, salaried, student, retired, entrepreneur, self-employed. |
| `credit_score` | integer | Кредитный скоринг клиента. Используется для score-сегментации и анализа DPD30+. |

---

## 2. Таблица `loan_applications`

Таблица содержит заявки клиентов на кредит.

| Поле | Тип данных | Описание |
|---|---|---|
| `application_id` | integer | Уникальный идентификатор заявки. Первичный ключ. |
| `client_id` | integer | Идентификатор клиента. Внешний ключ к `clients.client_id`. |
| `application_date` | date | Дата подачи заявки на кредит. |
| `product_name` | text | Название кредитного продукта, выбранного клиентом. |
| `requested_amount` | numeric | Запрошенная сумма кредита. |
| `status` | text | Статус заявки: approved, rejected, pending. |

---

## 3. Таблица `loans`

Таблица содержит только фактически выданные кредиты.

| Поле | Тип данных | Описание |
|---|---|---|
| `loan_id` | integer | Уникальный идентификатор кредита. Первичный ключ. |
| `application_id` | integer | Идентификатор заявки. Внешний ключ к `loan_applications.application_id`. |
| `client_id` | integer | Идентификатор клиента. Внешний ключ к `clients.client_id`. |
| `issue_date` | date | Дата выдачи кредита. |
| `product_name` | text | Название выданного кредитного продукта. |
| `loan_amount` | numeric | Фактически выданная сумма кредита. |
| `interest_rate` | numeric | Процентная ставка по кредиту. |
| `term_months` | integer | Срок кредита в месяцах. |

---

## 4. Таблица `payments`

Таблица содержит историю платежей по выданным кредитам.

| Поле | Тип данных | Описание |
|---|---|---|
| `payment_id` | integer | Уникальный идентификатор платежа. Первичный ключ. |
| `loan_id` | integer | Идентификатор кредита. Внешний ключ к `loans.loan_id`. |
| `payment_date` | date | Дата платежа. |
| `paid_amount` | numeric | Сумма платежа. |
| `overdue_days` | integer | Количество дней просрочки на дату платежа. Используется для расчёта DPD30+. |

---

## Связи между таблицами

| Связь | Тип связи | Описание |
|---|---|---|
| `clients.client_id` → `loan_applications.client_id` | one-to-many | Один клиент может подать несколько заявок. |
| `loan_applications.application_id` → `loans.application_id` | one-to-zero-or-one | Не каждая заявка становится кредитом. Кредит появляется только при фактической выдаче. |
| `clients.client_id` → `loans.client_id` | one-to-many | Один клиент может иметь несколько выданных кредитов. |
| `loans.loan_id` → `payments.loan_id` | one-to-many | Один кредит может иметь несколько платежей. |

---

## Аналитические примечания

- Основная аналитическая витрина строится на уровне заявки: одна строка соответствует одной кредитной заявке.
- Таблица `payments` перед соединением с основной витриной агрегируется до уровня `loan_id`, чтобы не задвоить заявки и кредиты.
- DPD30+ считается на уровне кредита: если максимальное значение `overdue_days` по кредиту больше или равно 30, кредит считается DPD30+.
- Основные риск-срезы в проекте: кредитный продукт, score-сегмент, тип занятости, vintage month и MOB.
- Для анализа кредитной воронки используются метрики `approval_rate`, `issue_rate` и `DPD30_rate`.
