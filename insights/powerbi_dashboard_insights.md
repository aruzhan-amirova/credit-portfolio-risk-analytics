# Power BI Dashboard Insights

Этот файл описывает основные выводы по страницам Power BI dashboard. Dashboard построен на основе подготовленных SQL views и помогает анализировать кредитную воронку, продуктовый риск, клиентские сегменты и vintage/MOB динамику.

---

## 1. Monthly Funnel

Страница `Monthly Funnel` показывает динамику кредитной воронки по месяцам:

- количество заявок;
- количество одобренных заявок;
- количество выданных кредитов;
- approval rate;
- issue rate;
- запрошенную и выданную сумму.

Основной смысл страницы — оценить, насколько стабильно работает кредитная воронка во времени.

По результатам анализа видно, что поток заявок по месяцам относительно стабилен. Approval rate находится примерно в диапазоне 58–68%, а issue rate в большинстве месяцев держится на уровне 70–90%.

Это означает, что значительная часть заявок получает одобрение, а большая часть одобренных заявок затем переходит в фактическую выдачу кредита.

---

## 2. Product Risks

Страница `Product Risks` показывает риск и объём выдач по кредитным продуктам:

- сумму выданных кредитов;
- количество кредитов DPD30+;
- сумму кредитов DPD30+;
- DPD30+ rate;
- DPD30+ amount share.

Эта страница помогает понять, какие продукты дают больший объём портфеля и какие продукты связаны с повышенным уровнем риска.

По результатам анализа наиболее рискованным продуктом выглядит `Micro loan`. Также повышенный DPD30+ rate наблюдается у `Cash loan`.

Это может указывать на более рискованный профиль клиентов в этих продуктах или на особенности самих продуктов: меньшие суммы, короткие сроки и более высокую чувствительность к платёжной дисциплине.

---

## 3. Client Segments

Страница `Client Segments` показывает различия между клиентскими score-сегментами:

- количество клиентов;
- средний credit_score;
- средний доход;
- сумма выдач;
- DPD30+ rate.

Основной вывод: сегмент `Low` credit_score показывает наиболее высокий уровень DPD30+.

Это подтверждает ожидаемую связь между кредитным скором и риском: клиенты с более низким credit_score чаще допускают просрочку 30+ дней.

Такой разрез помогает оценивать не только объём выдач по сегментам, но и качество этих выдач с точки зрения риска.

---

## 4. Vintage / MOB

Страница `Vintage / MOB` показывает развитие DPD30+ по месяцам выдачи кредита и месяцам жизни кредита.

Этот анализ важен, потому что свежие кредиты нельзя напрямую сравнивать со старыми: у новых кредитов ещё не было достаточно времени, чтобы выйти в просрочку 30+ дней.

Поэтому последние месяцы выдач могут показывать низкий DPD30+ rate не из-за улучшения качества портфеля, а из-за незрелости кредитов.

Vintage/MOB-матрица помогает корректнее сравнивать качество выдач, анализируя кредиты на одинаковом этапе их жизни.

---

## Итог по dashboard

Power BI dashboard позволяет смотреть на кредитный портфель с четырёх сторон:

- динамика кредитной воронки;
- риск по продуктам;
- качество клиентских сегментов;
- развитие просрочки через vintage/MOB.

Dashboard дополняет SQL-анализ и делает результаты более удобными для бизнес-мониторинга.
